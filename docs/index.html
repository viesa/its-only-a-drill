#item

To create an item:

CreateItem([The item you want to create], [The game Graphics], [The Enum of the item you want to create]);

AddToRenderList([The Game], [The item Drawable]);

To make the item stick to player:

UpdateItemPreview([The Player], [The Item])

#gui

Show alerts (prompts)

void AlertGui(Gui* g, int timer, char promptText[20]);

The prompt will show for 1/3 of timer. Animation timing is 1/3 slide in, 1/3 display, 1/3 slide out.
Recommended value is 2 (seconds)



Show messageBox

void MsgBoxShow(Gui* g, char message[201]);

Message boxes allow for showing more text than a prompt. The user can close the messageBox by clicking RETURN on their keyboard.
MessageBox also has a function to close, in case you desire to use it.

void MsgBoxHide(Gui* g);



Render text

void RenderText(Gui* g, int x, int y, int w, Color c, Format f, char text[]);

  • x: x Position, top left of text
  • y: y Position, top left of text
  • w: Width of text to print before new line. Set to 0 to deactivate
  • c: Color of text (Enum, use either int or color name)
    • 0 | Black
    • 1 | White
    • 2 | Red
    • 3 | Green
    • 4 | Yellow
    • 5 | Blue
    • 6 | Magenta
    • 7 | Cyan
  • f: Format of text (Enum, use either int or format name)
    • 0 | Regular
    • 1 | Bold
  • text: text to print

RenderText supports changing color and formatting inline.

Use the

$$FC

Format to change parameters. F being format as int, C being color as int.

#Eventhandeling

reads the events on your keyboard

creating a new event key first find the scancode for your key example 26 it's W.
Then give it a name that no other key has example UP1
last you need to add = so the finished new line looks like this "UP1= 26" in options.txt

eventhandler(["the event name"]) example eventhandler("UP1=")

the return value is 0 for false(the key was not pressed)
1 for true(the key was pressed)
